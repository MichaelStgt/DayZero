Project Requirements Specification

Phase 1: App Identity & Core Objectives

  • App Name: DayZero
  • Primary Goal: Provide a discrete, highly effective tool to help users initiate and maintain sobriety.

Phase 2: Functional Requirements (MVP)

  • Dynamic Core Tracking: Extensible data model supporting various addiction types (Alcohol, Nicotine, etc.).
  • Financial Tracking: Calculate money saved based on historical usage cost inputs.

Phase 3: Technical Architecture

  • Framework: .NET MAUI
  • Architecture Pattern: MVVM (MVVMTen Custom Library)

Phase 4: UI/UX & Navigation

  • Security: Biometric authentication required on app launch to protect sensitive user data.
  • First-Time User Experience (FTUE): A 4-step dynamic wizard:
    1. Substance Selection: Checkable, extensible list of habits to break.
    2. Date Selection: Setting the specific start date/time.
    3. Usage & Cost (Dynamic): Gathering financial baseline for future "Money Saved" metrics.
    4. Motivation: Setting the primary personal drive for quitting.
