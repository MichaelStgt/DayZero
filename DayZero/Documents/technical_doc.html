Technical Architecture & Documentation

1. Solution Structure

The project utilizes a strict MVVM pattern with singular folder naming conventions.

2. Navigation & Security

App startup and global navigation state are managed via a custom WindowCreator service.

  • Authentication: Enforces biometric checks before granting access to the main AppShell (bypassed in Debug mode).
  • Dynamic Route Registration: Views implementing IShellNavigationTarget are automatically registered to the MAUI Shell routing table at startup via reflection.
  • Onboarding Flow: A modular wizard structure utilizing route-parameter passing to construct the final SobrietyRecord.

View Interface Contract

To enforce architecture and enable auto-registration, Views implement the following contract:

public interface IShellNavigationTarget { ObservableViewModel? ViewModel { get; set; } }

Onboarding ViewModel Chain

The FTUE relies on a sequence of ViewModels passing state forward:

  • OnboardingSubstanceViewModel: Manages dynamic habit selection.
  • OnboardingDateViewModel: Captures the origin Date/Time.
  • OnboardingUsageViewModel: Captures financial/frequency metrics.
  • OnboardingMotivationViewModel: Finalizes the record and writes to local storage.
